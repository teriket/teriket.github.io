Project Narrative:

Introduction to the artifact

This is a full stack application developed in SNHU’s CS465: Full Stack Development I. It was developed between the months of June and August, 2025. This is a RESTful MEAN application with two components: a storefront that delivers statically generated html content to users looking to purchase vacation packages and an angular website for administrators to create, update, and delete vacation packages.

I've added analytics to the webpage. User page views will be recorded, and an analytics dashboard will be shown to admin users. I selected this artifact to reflect the growing trend of big data in computer science. Extracting information from user behaviors is a salient skill because recommender systems increase the value of a product to users which then generates revenue for a company. Business analysts also use this information to make important decisions. This also allows me to reflect in my portfolio the importance of ethics, data privacy, and secure coding practices in software engineering because extracting personally identifiable information touches on those three domains.

Enhancements Completed

The website now records when and which links users access. Administrators can see the total view count of each vacation package, as well as graphs of which packages have been viewed. This information is secured through an authentication service.

Reflection

There were two related challenges I faced while building this enhancement. I had to refamiliarize myself with the project architecture as well as the technology stack. One particular challenge I had to overcome was that data was not crossing over between HTML tags and javascript. This was caused by using uppercase letters in the data-* attribute. While I've read that using uppercase letters in html attributes is bad practice, I didn't know this was why. I overcame this challenge through experimenting with my code. Otherwise, what helped me in developing these features was to create incremental improvements, review past code I’ve written, and review API documentation.
